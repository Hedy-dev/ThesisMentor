# ThesisMentor

## Описание

ThesisMentor — система автоматизированной проверки текстов ВКР с генерацией рекомендаций по исправлению ошибок.

Текущий этап — реализация MVP (план минимум) в виде монолитного приложения с базовым ИИ-ядром.

Система решает задачи:
- структурирования текста ВКР;
- поиска типовых ошибок на основе правил и ML-моделей;
- генерации рекомендаций с использованием RAG;
- предоставления простого API для загрузки текста и получения результата.

---

## Основные возможности

- Загрузка текста ВКР
- Преобразование текста в структурированное представление (дерево)
- Поиск ошибок (модель "Критик")
- Генерация рекомендаций (модель "Генератор")
- Хранение результатов проверки
- Простой API / веб-интерфейс для демонстрации

## Архитектура (MVP)

Система реализована как монолит с упрощённой логикой взаимодействия компонентов:

- Одна SQL база данных (PostgreSQL)
- Асинхронные вызовы вместо брокеров сообщений
- Отсутствие Transactional Outbox
- Упрощённое хранение файлов (в БД или локально)

Основные модули:
- API слой (FastAPI)
- ИИ-ядро (парсинг, критик, генератор)
- База данных
- Векторная БД (Qdrant)

---

## Основные компоненты

### API (FastAPI)
- Загрузка текста
- Запуск проверки
- Получение результатов
- (опционально) авторизация пользователя

### Парсер текста
- Разбор документа ВКР
- Преобразование в древовидную структуру (JSON)
- Выделение секций, абзацев, заголовков

### RAG модуль
- Индексация внешних документов (методические материалы)
- Векторизация и загрузка в Qdrant
- Поиск релевантного контекста

### Модель "Критик"
- Набор правил для поиска ошибок
- Каждый тип ошибки — отдельный модуль
- Работает по структурированному представлению текста
- Формирует:
  - описание ошибки
  - позицию в тексте
  - формализованный контекст

Примеры типов ошибок:
- Несоответствие заголовка и содержимого
- Семантическая близость секций
- Повторы / дубликаты
- Структурные нарушения

### Модель "Генератор"
- Получает:
  - описание ошибки
  - контекст
- Выполняет:
  - поиск по векторной БД (RAG)
  - генерацию рекомендации (например, ruGPT)
- Обрабатывает кейс отсутствия релевантных данных

## Поток обработки

1. Пользователь отправляет текст
2. Текст парсится в структуру (JSON)
3. Модель "Критик" находит ошибки
4. Ошибки сохраняются в БД
5. Модель "Генератор" получает ошибки
6. Выполняется поиск по RAG
7. Генерируются рекомендации
8. Результат возвращается пользователю
